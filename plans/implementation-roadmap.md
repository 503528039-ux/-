# 雅洁五金 2026 - UI 重构实施路线图

## 1. 项目概览

### 1.1 项目目标
将现有单页应用重构为专业级三栏式编辑界面，应用 Quiet Luxury 设计语言，提升用户体验和开发维护性。

### 1.2 关键约束
- ✅ 保持与现有 Store 逻辑解耦
- ✅ 不修改 Command 管理逻辑
- ✅ 确保数据迁移系统兼容性
- ✅ 渐进式迁移，最小化风险

### 1.3 成功指标
- 页面加载性能提升 20%
- 代码可维护性提升（组件化程度）
- 用户满意度调查提升
- 开发效率提升（组件复用率）

## 2. 实施阶段划分

### Phase 0: 准备工作 (1-2天)
**目标**: 建立开发环境，创建基础架构

#### 任务清单:
- [ ] 创建组件目录结构
  ```
  src/components/layout/
  src/components/page/
  src/components/properties/
  src/components/ui/
  src/composables/
  ```
- [ ] 配置 TypeScript 类型定义
- [ ] 设置代码规范工具 (ESLint, Prettier)
- [ ] 创建组件模板和代码生成脚本
- [ ] 建立开发分支 `feature/ui-reconstruction`

#### 交付物:
- 完整的项目目录结构
- 类型定义文件
- 代码规范配置
- 组件模板

### Phase 1: 基础布局搭建 (3-4天)
**目标**: 实现三栏式布局框架，保持现有功能

#### 任务清单:
- [ ] 创建 `EditorLayout.vue` 布局容器
- [ ] 创建 `LeftSidebar.vue` 左侧导航
- [ ] 创建 `MainCanvas.vue` 主编辑区
- [ ] 创建 `RightSidebar.vue` 右侧面板
- [ ] 创建 `TopNavigation.vue` 顶部导航
- [ ] 集成现有 `App.vue` 内容到新布局
- [ ] 实现基础响应式布局
- [ ] 验证所有现有功能正常工作

#### 技术要点:
- 使用 CSS Grid/Flexbox 实现三栏布局
- 保持现有 Store 数据流不变
- 使用 `provide/inject` 进行组件通信
- 添加布局状态管理

#### 交付物:
- 可工作的三栏式布局
- 现有功能完整迁移
- 基础组件通信机制

### Phase 2: 视觉系统应用 (3-5天)
**目标**: 应用 Quiet Luxury 设计语言，优化视觉效果

#### 任务清单:
- [ ] 更新 Tailwind CSS 配置
  - 添加静奢色调色板
  - 配置字体系统
  - 定义阴影和圆角系统
- [ ] 应用玻璃质感效果到侧边栏

- [ ] 统一字体和间距系统
- [ ] 添加平滑过渡动画
- [ ] 实现暗色模式基础
- [ ] 优化图标和按钮样式

#### 技术要点:
- 使用 CSS 自定义属性定义设计 Token
- 实现 `backdrop-filter` 玻璃效果
- 使用 Tailwind 的 JIT 模式优化样式
- 添加 CSS 变量支持主题切换

#### 交付物:
- 完整的视觉设计系统
- 玻璃质感侧边栏
- 统一的组件样式
- 平滑的动画效果

### Phase 3: 功能模块提取 (4-6天)
**目标**: 将 App.vue 中的逻辑提取为可复用组合式函数
