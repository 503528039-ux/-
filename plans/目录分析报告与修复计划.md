## 🔍 关键问题识别

### 1. 组件映射逻辑问题
**问题描述**: 页面类型映射存在冗余和错误映射

**具体表现**:
- `App.vue`中的`pageComponents`映射表包含重复和临时映射
- `canvas`、`custom`、`brand`、`intro`类型都映射到`CatalogCover`组件
- 可能导致页面跳转逻辑混乱

**当前映射表**:
```javascript
const pageComponents = {
  'cover': CatalogCover,
  'companyIntro': CompanyIntroPage,
  'certificates': CertificatesPage,
  'partners': PartnersPage,
  'projectCases': ProjectCasesPage,
  'surfaceFinishes': SurfaceFinishesPage,
  'tableOfContents': TableOfContentsPage,
  'divider': DividerPage,
  'product': ProductGridPage,               // 6宫格产品页（门锁）
  'productGrid': ProductGridPage,           // 6宫格产品页（小五金）
  'compositeProduct': CompositeProductPage,
  'backCover': BackCoverPage,
  'free': FreePage,
  'canvas': CatalogCover,                   // 问题：临时映射
  'custom': CatalogCover,                   // 问题：临时映射
  'brand': CatalogCover,                    // 问题：临时映射
  'intro': CatalogCover                     // 问题：临时映射
}
```

### 2. 产品卡片视觉偏差问题
**问题描述**: `ProductCard.vue`组件未严格遵循原始设计稿的"Quiet Luxury"视觉标准

**设计规范要求**:
- 图片容器高度精确为`48mm`
- 产品图片使用`mix-blend-mode: multiply`效果
- 复合模式布局：实拍图58% + 线图38%
- 使用物理单位（mm）而非像素单位

**当前实现问题**:
- ✅ 图片容器高度已设置为`height: 48mm;`（第157行）
- ✅ 产品图片已使用`mix-blend-mode: multiply;`（第179行）
- ✅ 复合模式布局比例正确（58%/38%）
- ❌ 部分间距仍使用像素单位而非mm
- ❌ 复合模式边框和背景效果需要优化

### 3. 页面容器结构不统一
**问题描述**: 页面边距和容器结构不一致

**现状分析**:
- ✅ `A4Page.vue`已实现`15mm`物理边距（`.page-content { padding: 15mm; }`）
- ❌ 部分页面组件可能包含额外的内边距或容器
- ❌ 页面内容未完全限制在`A4Page`容器内

### 4. 代码冗余问题
**问题描述**: 存在重复和过时的代码文件

**具体表现**:
- `catalog.js`（27,032字符）和`catalog-new.js`（22,142字符）并存
- 需要确定哪个版本是最新、最完整的
- 清理未使用的代码和文件
