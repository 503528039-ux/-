# 雅洁五金 2026 图册重构计划

本计划旨在修复现有 Vue 3 项目中的视觉偏差和逻辑错误，使其回归原始设计精髓。

## 第一步：修正产品卡片视觉（Visual Regression）

**目标：** 精准还原 `ProductCard.vue` 的视觉样式，严格遵循“Quiet Luxury”设计稿。

1.  **读取原始设计稿：**
    *   分析 `页面代码.txt` 中第 79-89 行的 CSS 描述。
2.  **修改 `tailwind.config.js`：**
    *   在 `theme.extend.height` 中增加一个自定义值，用于 `48mm` 的图片容器高度。
    *   在 `theme.extend.spacing` 中增加 `15mm` 的页边距。
3.  **修改 `ProductCard.vue`：**
    *   将图片容器 `div` 的 class 替换为 `h-[48mm]`。
    *   确保图片使用 `object-cover` 和`mix-blend-multiply`。
    *   移除所有不符合设计稿的 Tailwind CSS 间距类，使用 CSS `gap` 或 `margin` 配合 `calc()` 函数实现精确控制。

## 第二步：修复页面跳转逻辑（Logic Fix）

**目标：** 解决产品页面错误跳转到封面的 Bug。

1.  **审查 `src/stores/catalog.js`：**
    *   检查 `pageComponent` 计算属性的逻辑。
    *   定位 `PageType.PRODUCT` 的映射关系。
2.  **审查 `src/types/catalog.ts`：**
    *   确认 `PageType` 枚举类型定义是否正确。
3.  **修复 `catalog.js`：**
    *   **关键操作：** 确保 `case PageType.PRODUCT:` 语句块明确返回 `ProductGridPage` 组件，而不是 `CatalogCover` 或其他任何组件。清除任何可能导致错误回退的逻辑。

## 第三步：统一页面容器结构（Structure Unification）

**目标：** 强制所有页面内容都在 `A4Page` 组件内渲染，并保持统一的 `15mm` 物理页边距。

1.  **审查 `App.vue` 或主布局文件：**
    *   检查路由视图 `router-view` 是否被 `A4Page` 组件包裹。
2.  **审查所有页面组件 (`src/components/pages/*.vue`)：**
    *   移除每个页面内部独立的布局容器或页边距样式。
    *   确保页面组件的根元素直接是内容，没有任何外层包裹。
3.  **修改 `A4Page.vue`：**
    *   使用 `p-[15mm]` Tailwind class 或在 `<style>` 标签中直接定义 `padding: 15mm`，来设置固定的物理页边距。

## 第四步：验证与切换

1.  **启动开发服务器**：运行 `npm run dev`。
2.  **视觉验证**：在浏览器中仔细比对 `ProductCard` 组件的渲染效果与原始设计稿。
3.  **功能验证**：点击 "门锁/五金" 按钮，确认页面正确跳转到产品列表，而不是封面。
4.  **切换模式**：在所有问题修复并验证通过后，切换到 `code` 模式，提交最终的全量代码。
