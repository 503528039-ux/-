# 控制组件与页面映射实施计划

## 🎯 核心需求理解

用户需求：确保本程序的控制组件（侧边栏等）插入页面时，对应的页面组件要与最新页面代码中的设计完全一致。

**关键点**：
1. 控制组件（如侧边栏按钮）需要正确映射到对应的页面组件
2. 每个页面组件需要与最新页面代码中的设计规范完全一致
3. 确保从控制组件创建页面的整个流程符合设计标准

## 🔧 技术实现细节

### 1. 页面组件结构优化
每个页面组件需要遵循以下结构：
```vue
<template>
  <A4Page>
    <!-- 统一的页眉 -->
    <PageHeader :title="pageTitle" />
    
    <!-- 页面内容区域 -->
    <div class="page-content">
      <!-- 具体页面内容，严格遵循设计规范 -->
    </div>
    
    <!-- 统一的页脚 -->
    <PageFooter :pageNumber="pageNumber" />
  </A4Page>
</template>
```

### 2. 设计规范提取和实现
从`最新页面代码.html`中提取CSS样式：
```css
/* 示例：荣誉资质页网格 */
.grid-cert {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 12mm;
  margin-top: 15mm;
  margin-bottom: 10mm;
}

.cert-box {
  border: 1px solid var(--divider);
  height: 75mm;
  border-radius: 6px;
  display: flex;
  align-items: center;
  justify-content: center;
  flex-direction: column;
  background: #fff;
}
```

### 3. 控制组件与页面数据流
```
侧边栏按钮点击
    ↓
创建页面数据 (store.createPageData)
    ↓
映射到对应组件 (App.vue pageComponents)
    ↓
渲染页面组件 (严格遵循设计规范)
    ↓
用户编辑和保存
```
