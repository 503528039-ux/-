**状态**: 🟡 有风险

**问题描述**:
`ImageUploader` **没有 `src` prop**，是纯上传触发覆盖层（`position: absolute; inset: 0`），空值回退完全由父组件负责，各页面均已实现 `|| placeholder` 兜底逻辑，组件本身不会产生乱码或崩溃。

1. 在组件注释中明确标注"父容器必须 `position: relative`"。

### 问题 3 — Store 数据流完整性

#### 3.1 insertPage 命令模式下数据丢失（严重）

**状态**: 🔴 有问题

**问题描述**:
`catalog-new.js` 的 `insertPage()` 在命令模式（`USE_COMMANDS = true`，当前默认开启）下存在 **数据丢失 bug**：

```js
// catalog-new.js:464-495
function insertPage(type, index = null) {
  const newPageData = getInitialPageData(type)  // ← 从 pageTemplates 深拷贝，但后续从未使用！
  // ...
  const command = createAddPageCommand({
    _addPage: (page) => { pages.value.splice(insertIdx, 0, page) },
  }, type)  // ← 只传 type 字符串，newPageData 被丢弃
```

`AddPageCommand.createPageData()` 内部自行创建数据，使用的是旧版类型名（`'cert'`、`'partner'`、`'finish'`），不认识新模板类型（`'certificates'`、`'partners'`、`'surfaceFinishes'`）。结果：任何通过侧边栏插入的新模板页，会走 `default` 分支，创建一个**只有 `type/id/blocks:[]/items:[]` 的空页面**，pageTemplates.js 中的所有预填数据（36 个工艺项、9 个案例项、8 个证书项等）被完全忽略。

**涉及文件**:
- `src/stores/catalog-new.js:464-495`
- `src/commands/page/AddPageCommand.ts:85-438`

**修复建议**:
在 `insertPage()` 命令模式分支中，将 `newPageData` 传入命令的 `pageData` 参数：
```js
const command = createAddPageCommand(
  { _addPage, _removePage, _movePage, _updatePage, getPages, setPages },
  type,
  undefined,    // subType
  newPageData   // ← 传入已深拷贝的页面数据
)
```
并在 `AddPageCommand.execute()` 第 87 行的判断中确保外部传入的 `pageData` 优先被使用。

#### 4.2 v-once 时序风险

**状态**: 🟡 有风险

`EditableText.vue:115` 的 contenteditable span 使用了 `v-once`：


Vue 不再管理该 DOM 节点的内容更新，外部 `props.value` 变化靠 `watch` 手动调用 `setText()` 修正。若 store 数据在 `onMounted` 之前已有异步更新（如 localStorage 恢复时机），`onMounted` 里的 `setText(props.value)` 可能读到过期值，`watch` 随后会修正，存在一帧闪烁的可能。

**涉及文件**: `src/components/ui/EditableText.vue:31-41、115`

### 问题 6 — CSS 冲突风险

#### 6.1 design-tokens.css 打印 padding 导致内容区收窄（严重）

**状态**: 🔴 有问题

`design-tokens.css:154-163` 的 `@media print` 中：

```css
@media print {
  .a4-page {
    padding: var(--size-page-padding);  /* = 15mm，四周均加 padding */
  }
}
```

此规则通过 `main.css:2` 的 `@import './design-tokens.css'` 生效，而 `main.css` 的 `@media print` 中没有对应的 padding 覆盖。

**实际影响**：打印时 `.a4-page` 获得 `padding: 15mm`（四周），而内部 `.page-content` 还有 `padding: 0 var(--page-padding)` = `0 15mm`。

→ A4 纸（210mm）的可用内容宽度：210 - 30×2 = **150mm**（规范应为 180mm）
→ **PDF 导出时所有页面内容区域比设计稿窄 30mm，版式错位。**

**涉及文件**: `src/assets/design-tokens.css:154-163`

**修复建议**:
```css
/* design-tokens.css — 删除或修改此段 */
@media print {
  .a4-page {
    /* 删除 padding 规则，由 .page-content 统一控制内边距 */
    padding: 0;
  }
}
```
